Заголовок: Некорректное начало телефона
Описание: Проверка того, что бэк не принимает некорректный телефон (некорректное начало Код города/оператора должен начинаться с цифры 3, 4, 5, 6, 8, 9)
Шаги:
1) Запусаем, например, Burp Suite
Результат - должен быть запущен Burp Suite
2) вводим http запрос на бэк сайта fintech.tinkoff.ru
Результат - мы должны видеть форму для тестов
3) Вводим значения для полей апи
Результат - верные поля
4) вводим несколько некорректных телефонов (222-222-22-22)
Результат - Бэк не должен пытаться записать данные в бд (ну скорее всего в бд) и должен сообщить нам об этом в своем ответе
